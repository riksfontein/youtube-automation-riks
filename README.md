# YouTube Automation System

Fully automated YouTube documentary production for Ancient Earth Cinema, The Global Intel Analyst, and The AI Bible Forensic.

---

## How It Works

**Four stages per video:**

| Stage | What happens | Trigger |
|-------|-------------|---------|
| 1 | Research 5 competitor videos, send Checkpoint 1 email | GitHub Actions schedule |
| 2 | Generate script from selected video, send script approval | You select video in GitHub |
| 3 | Generate 5 production documents, TTS, thumbnails | You approve script in GitHub |
| 4 | Assemble video + upload to YouTube | You upload assets and trigger in GitHub |

**Schedule:**
- Monday + Friday: Ancient Earth Cinema
- Tuesday + Saturday: The Global Intel Analyst
- Wednesday + Sunday: The AI Bible Forensic
- Thursday: Off

## Setup Instructions

### Step 1 — Add GitHub Secrets

Go to your repository → **Settings** → **Secrets and variables** → **Actions** → **New repository secret**

Add all 18 secrets below. The secret names must match exactly.

| Secret Name | Where to get it |
|---|---|
| `ANTHROPIC_API_KEY` | console.anthropic.com → API Keys |
| `ELEVENLABS_API_KEY` | elevenlabs.io → Profile → API Keys |
| `ELEVENLABS_VOICE_ID_AE` | Your custom AE voice ID from ElevenLabs |
| `ELEVENLABS_VOICE_ID_GIA` | Your custom GIA voice ID from ElevenLabs |
| `ELEVENLABS_VOICE_ID_BF` | Your custom BF voice ID from ElevenLabs |
| `SUBSCRIBR_API_KEY` | Subscribr dashboard → Settings → API |
| `SUBSCRIBR_CHANNEL_ID_AE` | Your AE channel ID in Subscribr |
| `SUBSCRIBR_CHANNEL_ID_GIA` | Your GIA channel ID in Subscribr |
| `SUBSCRIBR_CHANNEL_ID_BF` | Your BF channel ID in Subscribr |
| `VIDIQ_API_KEY` | Your vidIQ API key |
| `RESEND_API_KEY` | resend.com → API Keys |
| `RESEND_FROM_EMAIL` | The email address automation sends FROM |
| `RESEND_TO_EMAIL` | The email address you receive notifications at |
| `YOUTUBE_CLIENT_ID` | Google Cloud Console → APIs & Services → Credentials |
| `YOUTUBE_CLIENT_SECRET` | Google Cloud Console → APIs & Services → Credentials |
| `YOUTUBE_REFRESH_TOKEN_MAIN` | Generated by running youtube_drive_auth.py |
| `GOOGLE_DRIVE_FOLDER_ID` | The folder ID of your YouTube-Automation folder in Drive |
| `GITHUB_PAT` | github.com → Settings → Developer settings → Personal access tokens |

### Step 2 — Create a GitHub Personal Access Token

This allows approval emails to trigger the next workflow stage.

1. Go to github.com → click your profile picture → **Settings**
2. Scroll to the bottom → **Developer settings**
3. **Personal access tokens** → **Tokens (classic)**
4. Click **Generate new token (classic)**
5. Name: `YouTube Automation`
6. Expiration: No expiration
7. Scopes: check **workflow** and **repo**
8. Click **Generate token** — copy it immediately
9. Add it as the `GITHUB_PAT` secret

### Step 3 — Create the Google Drive folder structure

1. Open Google Drive
2. Create a folder named `YouTube-Automation`
3. Inside it, create subfolders: `AE`, `GIA`, `BF`
4. Inside each channel folder, create: `reference-library`
5. Copy the folder ID from the URL: `drive.google.com/drive/folders/THIS_PART_IS_THE_ID`
6. Add it as the `GOOGLE_DRIVE_FOLDER_ID` secret

### Step 4 — Upload reference images

Upload your Nano Banana Pro composite images and GPT Image 2 map/creature images to the correct folders in Google Drive:
- `YouTube-Automation/AE/reference-library/`
- `YouTube-Automation/GIA/reference-library/`
- `YouTube-Automation/BF/reference-library/`

Then update `state.json` with the Google Drive file URLs for each reference image in the `reference_library` sections.

### Step 5 — Enable GitHub Actions

Go to your repository → **Actions** → click **Enable Actions** if prompted.

## Running a Test

To test without waiting for the schedule:

1. Go to **Actions** → **Stage 1 — Research & Competitor Selection**
2. Click **Run workflow**
3. Select channel: `AE`
4. Click **Run workflow**

You should receive a Checkpoint 1 email within 5 minutes.

## How to Respond to Emails

**Checkpoint 1 email (competitor video selection):**
1. Review the 5 videos in the email
2. Click **SELECT THIS VIDEO →** on your chosen video — opens GitHub Actions
3. Click **Run workflow**, fill in: channel, job_id, video URL, video title
4. Click Run

**Checkpoint 2 email (script approval):**
1. Read the script
2. Click **APPROVE** or **REGENERATE** — opens GitHub Actions
3. Click **Run workflow**, fill in: channel, job_id, action (approve or regenerate)
4. Click Run

**Documents email:**
1. Download all 5 attached Word documents
2. Generate images using Doc 2 in Google Flow (VEO Automation extension)
3. Generate videos using Doc 3A (Meta AI), Doc 3B (VEO3.1), Doc 3C (Grok Aurora)
4. Upload images to the `images/` subfolder in the Drive folder shown in the email
5. Upload videos to the `videos/` subfolder
6. Files must be named: `001_image.jpg`, `002_image.jpg` etc. and `001_video.mp4`, `002_video.mp4` etc.
7. Click **ASSETS READY** — opens GitHub Actions
8. Fill in: channel, job_id, Drive folder URL
9. Click Run

## Adding New Channels (Elite Chronicles etc.)

When a new channel is ready:
1. Add a new entry to `state.json` with the channel config
2. Add a `schedule_days` field
3. Update the cron in `stage1_research.yml`
4. Add the new voice ID as a secret: `ELEVENLABS_VOICE_ID_EC`
5. Add the Subscribr channel ID as a secret: `SUBSCRIBR_CHANNEL_ID_EC`

## Troubleshooting

**Research returns no videos:** Check that the vidIQ and NexLev API connections are working. Try running Stage 1 manually.

**Script generation times out:** Subscribr can take up to 8 minutes. If it fails, re-run Stage 2 with the same job ID and action = regenerate.

**Assembly fails:** Confirm video clips are named `001_video.mp4` format and all scene numbers are present. Check GitHub Actions logs.

**YouTube upload fails:** Verify the OAuth refresh token is valid by running `youtube_drive_auth.py` again.
